## 💰 Engenharia de Receita (Revenue Engineering)

Um projeto moderno deve ser sustentável. Implementei uma lógica de negócios invisível mas potente:

* **Affiliate Strategy (Amazon):** No catálogo da página `LivrosDoados`, integramos livros tipo `affiliate`. Isso transforma a curiosidade do usuário em conversão direta. A UX mantém a confiança, enquanto o sistema gera faturamento passivo.
* **Self-Sponsorship (Patrocínio Próprio):** O tráfego do projeto é direcionado para minha agência e portfólio. É a aplicação do conceito de **CAC Zero** (Custo de Aquisição Zero) para escalar minha marca pessoal.

## 🧩 Análise Técnica dos Componentes (O "Porquê" e "Para Quê")

### 1. `Inicio.jsx` (Conversão & Retenção)
- **O que foi feito:** Hero Section com fundo de vídeo dinâmico, Parallax e Cards 3D.
- **Porquê:** O uso de `rotateX` e `rotateY` via `framer-motion` cria uma sensação de materialidade física. Isso aumenta a percepção de valor do conteúdo "Premium", retendo o usuário por mais tempo (stickiness).
- **Termos Técnicos:** 3D Transforms, useScroll, useMotionValue, Hardware Acceleration.

### 2. `LivrosDoados.jsx` (Arquitetura de Dados Híbrida)
- **O que foi feito:** Grid interativo de livros que mescla doações físicas (API), PDFs gratuitos (Catálogo Local) e Ofertas Amazon (Afiliados).
- **Porquê:** O uso de `useSearchParams` permite que qualquer filtro de busca seja "Deep Linked" (compartilhável por URL). A lógica híbrida garante que o usuário sempre encontre valor imediato (PDF) ou suporte o projeto (Afiliados).
- **Termos Técnicos:** Stateful Routing, Debounce Search, Skeleton Loading, Logical Merging.

### 3. `QueroDoar.jsx` (Redução de Fricção via AI)
- **O que foi feito:** Formulário com "Magic Fill" que extrai metadados de descrições livres e busca capas automaticamente via Google Books API.
- **Porquê:** A maior barreira para doações é o preenchimento de formulários longos. A IA resolve isso, aumentando drasticamente a taxa de doação (conclusão de meta).
- **Termos Técnicos:** AI Orchestration, Entity Extraction, API Integration, Auto-Save/LocalStorage Persistence.

### 4. `useBooks.js` (Resiliência & UX de Confiança)
- **O que foi feito:** Hook customizado com lógica de **Fallback Híbrido** e Modo Offline.
- **Porquê:** Para garantir uma experiência de 100% de disponibilidade. Se o backend falha, a UI ativa o modo fallback instantaneamente, mantendo a confiança do usuário no produto.
- **Termos Técnicos:** Custom Hooks, Resilience Patterns, Offline First, Conditional Fetching.
